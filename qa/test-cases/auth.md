# Тест-кейсы: Авторизация

**Модуль:** Auth  
**Base URL:** `http://localhost:3001/api` (Docker) / `http://localhost:5000/api` (локально)  
**Версия:** 1.0  
**Дата:** 2026-06-08

---

## TC-AUTH-001 — Регистрация с валидными данными

| Поле | Значение |
|------|----------|
| **Приоритет** | High |
| **Предусловия** | Email `qa001@taskflow.test` не зарегистрирован в системе |

**Шаги:**
1. Отправить `POST /api/auth/register` с телом:
   ```json
   {
     "email": "qa001@taskflow.test",
     "username": "qa_user_001",
     "password": "SecurePass123!",
     "fullName": "QA User 001"
   }
   ```

**Ожидаемый результат:**
- HTTP `201 Created`
- `success: true`
- В `data` присутствуют `user` (id, email, username, role) и `token` (непустая строка)
- Пользователь сохранён в БД

## TC-AUTH-002 — Регистрация с уже существующим email

| Поле | Значение |
|------|----------|
| **Приоритет** | High |
| **Предусловия** | Пользователь с email `existing@taskflow.test` уже зарегистрирован |

**Шаги:**
1. Зарегистрировать пользователя с email `existing@taskflow.test` (если ещё не создан).
2. Повторно отправить `POST /api/auth/register` с тем же email и другим username.

**Ожидаемый результат:**
- HTTP `400` или `409`
- `success: false`
- Сообщение об ошибке указывает на занятый email
- Новый пользователь не создаётся

## TC-AUTH-003 — Логин с верным паролем

| Поле | Значение |
|------|----------|
| **Приоритет** | High |
| **Предусловия** | Пользователь `login_ok@taskflow.test` зарегистрирован с паролем `SecurePass123!` |

**Шаги:**
1. Отправить `POST /api/auth/login`:
   ```json
   {
     "email": "login_ok@taskflow.test",
     "password": "SecurePass123!"
   }
   ```

**Ожидаемый результат:**
- HTTP `200 OK`
- `success: true`
- В `data` присутствуют `user` и `token`

## TC-AUTH-006 — Доступ с просроченным токеном

| Поле | Значение |
|------|----------|
| **Приоритет** | Medium |
| **Предусловия** | Имеется JWT с истёкшим `exp` (сгенерирован с коротким TTL или подставлен заранее) |

**Шаги:**
1. Отправить `GET /api/auth/me` с заголовком `Authorization: Bearer <expired_token>`.

**Ожидаемый результат:**
- HTTP `401 Unauthorized`
- `success: false`
- Сообщение о невалидном или просроченном токене
